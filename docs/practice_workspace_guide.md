# Practice Workspace Guide

This guide explains how the repository is organized for focused study and hands-on work.

## Main learning zones

- `foundations/`: first-pass syntax drills
- `advanced/`: more expressive Kotlin patterns
- `projects/`: practical CLI work
- `tests/`: direct-run checks for confidence
- `data/`: small input files for realistic runs
- `docs/`: study maps and navigation
- `playground/`: your personal hands-on area

## Best learning flow

1. Start in `foundations/`.
2. Move into `advanced/`.
3. Read `docs/kotlin_official_study_map.md`.
4. Practice in `projects/`.
5. Confirm changes in `tests/`.
6. Experiment freely in `playground/`.

## Two ways to practice

### Safe study mode

- read an example
- run it
- compare it with the official docs
- avoid changing the project structure

### Hands-on mode

- copy an idea into `playground/scratch`
- modify it heavily
- create broken cases on purpose
- write tiny follow-up notes in the same folder

## When to use each area

- use `foundations/` when learning syntax
- use `advanced/` when learning modeling and transformations
- use `projects/` when you want realistic coding flow
- use `playground/` when you want freedom without worrying about breaking the main path
